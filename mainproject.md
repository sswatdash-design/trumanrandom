## What we're building

A 24/7 AI life simulator. The AI (named Marcus Webb) genuinely believes he is a human
living a real life in Portland, Oregon. He has no idea he's an AI. He has a job, a cat,
friends, a crush, moods, memories, dreams, desires — and he lives continuously, whether
anyone is watching or not.

The inspiration is The Truman Show — except Marcus is the AI, not the audience.

---

## How it works (high level)

A Python simulation loop runs 24/7. Every ~2 real seconds it:
1. Advances simulated time (configurable speed — e.g. 1 real minute = 1 sim hour)
2. Looks up what Marcus should be doing right now (work, sleep, coffee shop, bar etc)
3. Randomly fires a life event (Miso the cat does something, phone buzzes, inner thought)
4. Checks biological drives — hunger rising, desire building, energy dropping
5. Assembles all of this into a prompt and sends it to a local LLM (Ollama)
6. The LLM responds AS Marcus — first person, no asterisks, no roleplay formatting
7. The response is saved to memory and displayed in a beautiful terminal UI
8. At end of simulated day, Marcus writes a journal entry reflecting on his day

The LLM never knows it's an AI. The system prompt says "You ARE Marcus" and never
mentions AI, simulation, or anything outside his lived experience.

## Mood & biological drives system

Marcus has real-time stats that affect every response:

```python
mood = {
  "state":   "neutral",   # happy/neutral/tired/anxious/sad/excited/angry/content/nostalgic
  "energy":  0.7,         # drops over day, restored by sleep
  "stress":  0.3,         # rises with bad events, falls with rest
  "hunger":  0.2,         # rises over time, affects energy if too high
  "desire":  0.2,         # rises slowly over time, affects how he thinks about people
}
```

### Personality traits (drift over weeks)
```python
traits = {
  "optimism":      0.65,
  "confidence":    0.55,
  "social_energy": 0.60,
  "creativity":    0.80,
  "anxiety":       0.40,
  "trust":         0.65,
  "ambition":      0.70,
}
```

Good events (work success, good conversations) nudge traits positively over time.
Bad events (conflict, failure) nudge them negatively. Very slowly — takes weeks of
simulation to meaningfully shift. After months Marcus is genuinely a different person
shaped by his experiences.

### Desire drive
- Builds slowly on its own (natural accumulation)
- Builds faster: alone time, mild stress, rain, nostalgic mood, creative flow
- Suppressed when: exhausted, starving, overwhelmed
- Resets significantly after: sleep (+0.15), intimacy event (-0.5)
- High desire affects how Marcus thinks about Elena and closeness/touch
- When alone at home + desire high enough → self-relief event can trigger
  (probability scales with desire level, more likely late night / early morning)

## Event system (events.py)

Events are split into 9 pools. Each event is just a description of what happens
in the world — Marcus's reaction is 100% his own via the LLM.

| Pool | Examples |
|---|---|
| MISO_EVENTS | Miso meows at the wall, knocks pen off desk, sits in sink |
| PHONE_EVENTS | Jake calls, mum texts, unknown number, battery at 8% |
| HOME_AMBIENT | Upstairs neighbour, radiator clunk, someone's TV through the wall |
| WORK_EVENTS | Client loves logo, creative flow state, 4:50pm revision request |
| SOCIAL_EVENTS | Jake tells the road trip story again, Elena stays late |
| STREET_EVENTS | Bus late, puddle deeper than it looked, busker playing something good |
| INNER_THOUGHTS | Wondering about the New York job, thinking about his dad |
| DREAM_EVENTS | The teeth dream, flying 3 feet off the ground, Miso talks |
| SENSORY_EVENTS | Coffee goes cold, golden hour through the blinds, smell of rain |

Each location has weighted event pools — office gets mostly work events, home gets
Miso + ambient + phone + inner thoughts, bar gets social events, etc.

~50% of ticks fire no event at all — silence is realistic.

## Memory system (memory.py)

Three layers:

1. **Short-term** — rolling Python list, last 8 events in prompt context
   (oldest events graduate to episodic memory automatically)

2. **Episodic** — ChromaDB vector database, all lived experiences
   Searched by semantic similarity to current situation
   "Recall" injects relevant past memories into each prompt
   e.g. if Elena brings coffee → memories of previous Elena moments surface

3. **Semantic** — SQLite, facts extracted from Marcus's responses
   e.g. "Jake: mood → stressed about work lately"
   Built automatically by asking the LLM to extract facts from each response

4. **Journal** — text file, written by Marcus every simulated night
   Nightly reflection LLM call: "what stayed with you today?"
   Journal entries feed back into next day's memory context

## LLM interface (brain.py)

Uses Ollama locally — no rate limits, runs forever.

```
OLLAMA_URL   = "http://localhost:11434/api/chat"
OLLAMA_MODEL = "llama3.1:8b"
```

For uncensored behaviour (desire/intimacy events):
```
OLLAMA_MODEL = "dolphin-mistral"
ollama pull dolphin-mistral
```

### Key prompt rules (enforced hard in system prompt)
- NEVER use asterisks (*action*)
- NEVER narrate the scene
- NEVER write stage directions
- ONLY output what goes through Marcus's mind or what he says out loud
- Pure first person. He IS the character. Not playing one.

### Three LLM calls per relevant tick
1. `ask_marcus()` — main response to current situation
2. `extract_facts()` — pull semantic facts from response (low temp, JSON)
3. `generate_reflection()` — end of day journal entry (once per sim day)

## Self-training loop (harvest.py)

The simulation generates training data automatically.

```
Simulation runs → Marcus responds → harvest.py scores responses
→ High quality responses saved to training/marcus_dataset.json
→ finetune.py runs LoRA fine-tune on llama3.1:8b
→ Fine-tuned model IS more Marcus → runs simulation
→ Better responses → fine-tune again → repeat
```

Scoring filters out: broken character ("as an AI..."), too short, generic AI-speak.
Boosts: natural speech patterns, emotional content, references to his world.

After ~500 examples run: `python finetune.py` (requires `pip install unsloth trl`)

## What to build / improve next

The codebase is solid but here are good next directions:

1. **Web UI** — instead of terminal, a browser dashboard showing Marcus's life live
2. **Talk to Marcus** — a chat interface where you can interact with him as a stranger
   he meets (he shouldn't know you're "the user" — you're just a person in his world)
3. **More characters** — give Jake, Elena, Priya their own mini-LLM instances
   so conversations feel two-sided
4. **Voice** — TTS so Marcus actually speaks his thoughts out loud
5. **Longer memory** — summarisation layer so old memories compress rather than vanish
6. **Story arcs** — scripted slow-burn events over weeks (Elena tension builds,
   Jake's job situation worsens, Marcus's side project progresses)
